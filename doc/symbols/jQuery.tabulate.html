<static> jQuery.tabulate.apply_properties(jQuery, Object, properties)
Applies properties to rows, columns and cells in the table.

Each row or column in the table may have any number of properties assigned to it. These properties are passed into the plugin upon initialization as jQuery.tabulate.options.rows for rows, and jQuery.tabulate.options.columns for columns. The following outlines valid properties for both rows and columns (note that "cell" refers to a column within a row):

  • name - The name associated with the current cell or row. In the case of an object, this would be the key. In the case of an Array, this would be the index.
  • data - The data associated with the current row. It can be used for extracting the value of a key and storing it as the cell's content.
  • args - An array of arguments to pass to the callback function via javaScript's apply function.

  • callback - This is a function that will be called when the cell or row is instantiated. This is useful for manipulating something based on the data assigned to a cell or row. Inside of the this function, this is equal to jQuery.tabulate. The arguments passed to this function vary between cells and rows.

    The arguments passed for a row are:

    • $row - The jQuery object representing the current row.
    • row_data - The data associated with the current row.

    The arguments passed for a cell are:

    • $row - The jQuery object representing the current row.
    • $cell - The jQuery object representing the current cell.
    • $content - The jQuery object representing $cell's content wrapper.
    • row_data - The data associated with the current row.
  • event_handlers - An object containing event handlers to bind to the cell or row. These should be defined in the same format as jQuery.tabulate.event_handlers.
  • styles - An object containing style information to be applied to the row or cell. This object is passed to the jQuery.css function.

Columns may also contain these properties, in addition to those defined above:

  • $content - The jQuery object in which to append content inside of. If not given, defaults to $element.
  • content - The content to append inside $content. Content can be passed in as a string, a jQuery object, or as a key within the dataset data. If you wish to pass in a key, you may do so within curly braces like so: "{some_key}". If the key is nested within the dataset, you may separate each level by use of a dot like so: "{obj.prop.key}".

<static> jQuery.tabulate.gather_data(Object, Object)
Gathers the data needed for tabulation.

Data can either be passed in explicitly or loaded via jQuery.ajax. This function expects a "request" parameter, however, if none is passed it will use the default data request object that was passed into tabulate upon initialization.

If a request is found, this function will trigger jQuery.tabulate.event_handlers.event:loading with true as its argument, effectively turning it on. After the data type is determined and the data has been gathered this function will trigger jQuery.tabulate.event_handlers.event:post_load with the data object as its argument. It is important to note that if both an AJAX request and a JSON object are passed to this function, the result of the two will be merged together before being passed to the post_load event handler.

Parameters:
Object
request The request object.
Object
filters Filters that will apply to this request only.

<static> jQuery.tabulate.update_filters(Object, Object)
Updates the filters that will be applied to the current dataset.

This function will merge any filters that are passed in with those already present, overwriting any values that are already contained in the filters Object. Any type of filter may be passed in here, including custom filters that you wish to pass to your controller.

When the merge is complete, the current_page will be reset to 1 and the jQuery.tabulate.event_handlers.event:refresh event handler is called which reloads the data with the new filters applied.

Parameters:
Object
filters The filters to apply to the dataset.
Object
refresh Whether or not to call the refresh handler. Defaults to true.
